# SPACE AUDIO for YMM4

[![License](https://img.shields.io/badge/license-MIT-blue.svg)](#)
[![.NET](https://img.shields.io/badge/.NET-10.0-purple.svg)](#)
[![Release](https://img.shields.io/github/v/release/routersys/YMM4-SpaceAudio.svg)](https://github.com/routersys/YMM4-SpaceAudio/releases)

YukkuriMovieMaker4 (YMM4) 上で動作する、物理音響モデルに基づく**ルームリバーブシミュレーター音声エフェクトプラグイン**です。
初期反射エンジン・フィードバックディレイネットワーク・素材吸収係数・音源/リスナー配置といった要素を組み合わせ、仮想空間における残響を多角的にシミュレートします。
さらに、インタラクティブな3Dルームエディタ・プリセット管理・YMM4アニメーション完全統合を備え、映像制作における音声演出の幅を大幅に広げます。

![SPACE AUDIO Screenshot](https://github.com/routersys/YMM4-SpaceAudio/blob/main/docs/SpaceAudio.png)

## 概要

本プラグインは、YMM4の音声エフェクトとして動作するルームリバーブシミュレーターです。
音源とリスナーを3D空間内に配置し、部屋の寸法・形状・壁素材・残響パラメータを組み合わせることで、
実空間に近い残響感を音声に付加できます。

> **【3Dビューとオーディオ処理について】**<br>
> プロパティパネルに表示される3Dルームビューは、設定内容を**視覚的に把握するための表示**です。<br>
> 実際の音声処理は、部屋の寸法・音源/リスナー座標・吸収係数・残響時間などのパラメータ値に基づいて行われます。<br>
> ルーム形状は初期反射パターンとFDNのチューニングに影響しますが、<br>
> 幾何学的な音線追跡（レイトレーシング）による完全な物理シミュレーションではありません。<br>
> 3D表示上の形状はあくまで「意図した空間のデザインを視覚的に伝える」ものとして活用してください。<br>

全パラメータはYMM4のアニメーションシステムと完全に統合されており、
部屋の寸法・音源位置・残響時間にいたるまで、タイムライン上でキーフレームアニメーションを設定できます。

## 主な機能

### 1. 物理音響モデルに基づくリバーブエンジン

初期反射と後期残響を独立したエンジンで処理し、2段階のリバーブ構造を実現します。

- **初期反射エンジン**: 最大24本の一次反射パスを計算し、
  音源・リスナー・部屋の寸法から各反射のディレイ時間と減衰量・パンニングを算出します。
- **フィードバックディレイネットワーク**: 8ラインのFDNとアダマール行列混合による
  高密度な後期残響を生成します。素数ディレイ長を採用し、音楽的に自然な音色を実現しています。
- **プリディレイ**: 0〜200msのプリディレイにより、直接音と反射音の時間的分離を調整できます。
- **HFダンピング**: 2段カスケード一次ローパスフィルターにより、高域の空気吸収をシミュレートします。
- **拡散**: オールパスフィルターのゲインを制御し、残響の拡散度を調整します。

### 2. 4種類のルーム形状

ルーム形状の選択は3Dビューの見た目だけでなく、初期反射の生成パターンとFDNのスケーリングに影響します。

| 形状 | 説明 |
|---|---|
| 矩形 | 標準的な直方体。汎用的なルームリバーブに最適です。 |
| L字型 | 間仕切りのある空間を想定。非対称な反射パターンが生まれます。 |
| 大聖堂 | 高い天井と長い残響を持つ大型空間。スケールと拡散が増強されます。 |
| スタジオ | 吸音処理された閉鎖空間を想定。残響が短く締まった音になります。 |

### 3. 8種類の壁素材

壁・床・天井それぞれに異なる素材を設定できます。素材は音響吸収係数として処理され、
初期反射の減衰量と後期残響の減衰時間に直接影響します。

| 素材 | 吸収係数 | 特性 |
|---|---|---|
| コンクリート | 0.02 | 硬質、反射が強い |
| 木材 | 0.10 | 自然な響き |
| ガラス | 0.03 | 鋭い反射 |
| カーペット | 0.50 | 中域を吸収 |
| 吸音パネル | 0.80 | 強い吸音 |
| レンガ | 0.05 | やや硬質 |
| 石膏ボード | 0.12 | 一般的な室内壁 |
| タイル | 0.02 | 硬質、浴室的な響き |

### 4. インタラクティブな3Dルームエディタ

ソフトウェアレンダリングによる3Dビューを搭載しています。パースペクティブ投影と近平面クリッピングを実装し、
任意の視点から部屋・音源・リスナーの配置を視覚的に確認・編集できます。

- **カメラ操作**: 左ドラッグで回転・右ドラッグでパン・マウスホイールまたはボタンでズームを行えます。
- **音源/リスナーの直接操作**: 3D空間内の「S」(Source)・「L」(Listener) マーカーを直接ドラッグして位置を調整できます。
- **視点プリセット**: Top（上面）・Front（正面）・Side（側面）・Reset（初期視点）ボタンで素早く視点を切り替えられます。
- **ルーム情報オーバーレイ**: 3Dビュー左上に部屋の寸法・体積・音源〜リスナー距離・RT60・ドライ/ウェット比をリアルタイム表示します。
- **軸インジケーター**: ビュー左下にX・Y・Z軸インジケーターを表示し、現在のカメラ向きを把握できます。
- **リサイズ**: エディタ右下のドラッグハンドルで高さを80〜800pxの範囲で変更でき、設定値は永続化されます。

### 5. 音源 / リスナー配置

- 音源とリスナーはそれぞれX・Y・Z座標で配置します（各軸が部屋の寸法を超えないようにクランプされます）。
- 音源〜リスナー間の距離は反射の到達時間・方向・ゲインに影響します。
- 各座標はYMM4のアニメーションシステムに対応しており、音源が動くような演出も可能です。

### 6. リバーブパラメータ

| パラメータ | 範囲 | 説明 |
|---|---|---|
| Pre-Delay | 0〜200 ms | 直接音から初期反射までの遅延時間 |
| Decay Time (RT60) | 0.1〜10 s | 残響の減衰時間 |
| HF Damping | 0〜1 | 高周波の減衰量（空気吸収のシミュレーション） |
| Diffusion | 0〜1 | 残響の拡散度（エコー密度） |
| Early Level | -40〜+6 dB | 初期反射音のレベル |
| Late Level | -40〜+6 dB | 後期残響音のレベル |
| Dry/Wet | 0〜1 | 原音とエフェクト音の混合比 |

### 7. アニメーション完全統合

全パラメータはYMM4のアニメーションシステムと統合されています。

- アニメーションが設定されている場合、フレームごとにパラメータを補間し、
  リアルタイムにリバーブエンジンを再設定します。
- アニメーションが設定されていない場合はブロック単位で処理され、不要な再設定コストを回避します。

### 8. プリセット管理システム

現在のルーム設定をプリセットとして保存・読み込みできます。

- プリセットファイルは `SARP` マジックバイト + CRC32チェックサム付きのバイナリ形式（`.sap`）で保存されます。
- データ破損時は `.bak` ファイルからの自動復元を試みます。
- 新規エフェクト追加時に自動適用するデフォルトプリセットを設定画面から指定できます。

### 9. 処理品質設定

設定画面から エコノミー / スタンダード / 高品質 の3段階の処理品質を選択できます。

### 10. 出力保護

後段処理として以下の保護機構を備えています。

- **ソフトクリッパー**: 三次歪みによる緩やかなクリッピングで、急峻な波形の歪みを防ぎます。
- **ピークリミッター**: アタック0.1ms・リリース50msのエンベロープフォロワーにより、
  出力レベルが閾値を超えた場合にゲインを自動制御します。
- **ステレオワイドナー**: ミッド/サイド処理によるステレオ幅の調整を行います。

### 11. エフェクトラベルの動的表示

タイムライン上のエフェクトアイテムのラベルが現在の設定を反映してリアルタイムで更新されます。

```
スペースオーディオ - {幅}x{高さ}x{奥行き}m RT60:{残響時間}s
```

## 動作要件

- **OS**: Windows 10 / 11 (64bit)
- **YukkuriMovieMaker4**: 最新版を推奨
- **ランタイム**: .NET 10.0

## インストール方法

1. [リリースページ](https://github.com/routersys/YMM4-SpaceAudio/releases/latest)から最新の `SpaceAudio` プラグインファイル（`.ymme`）をダウンロードしてください。
2. YMM4が起動していないことを確認し、ダウンロードしたファイルを実行（ダブルクリック）してインストールします。
3. YMM4を起動し、音声エフェクトの追加ダイアログを開きます。
4. **「エフェクト」グループ**の中に **「スペースオーディオ」** が追加されていますので、それを選択してください。

## 使い方ガイド

### 基本的な使用フロー

1. YMM4のタイムライン上で音声アイテムを選択し、エフェクトの追加を行います。
2. **「エフェクト」グループ**から **「スペースオーディオ」** を選択します。
3. エフェクトのプロパティパネルに3Dルームエディタが表示されます。
4. 部屋の寸法・形状・素材をパラメータパネルで設定します。
5. 3Dビュー上で「S」マーカー（音源）と「L」マーカー（リスナー）をドラッグして配置します。
6. 残響パラメータを調整し、目的の空間感を作ります。
7. Dry/Wet ミックスで原音とエフェクト音のバランスを設定します。

### 3Dルームエディタ 操作一覧

| 操作 | 動作 |
|---|---|
| 左ドラッグ（空間部分） | カメラを回転 |
| 右ドラッグ | カメラをパン（平行移動） |
| マウスホイール | ズームイン・アウト |
| 「S」マーカーをドラッグ | 音源の位置を変更 |
| 「L」マーカーをドラッグ | リスナーの位置を変更 |
| T ボタン | 上面図に切り替え |
| F ボタン | 正面図に切り替え |
| S ボタン | 側面図に切り替え |
| R ボタン | ビューをリセット |
| +/− ボタン | ズームイン・アウト |
| エディタ右下をドラッグ | エディタの高さをリサイズ |

### プリセットの使い方

1. 3Dルームエディタ上部のドロップダウンをクリックします。
2. 一覧から適用したいプリセットをダブルクリックします。
3. 現在の設定を保存するには「保存」ボタンをクリックし、プリセット名を入力してください。
4. プリセットの削除・名前変更は右クリックのコンテキストメニューから行えます。

### 設定画面

歯車アイコンから開く設定画面では、以下の項目を変更できます。

- **処理品質**: エコノミー / スタンダード / 高品質 を選択します。
- **デフォルトプリセット**: 新規エフェクト追加時に自動で読み込むプリセットを指定します。
- **プリセット管理**: プリセットの一覧確認・削除が行えます。

## 注意事項

- **3Dビューはデザイン補助ツールです**: 3D表示上の部屋の形は設定したパラメータを視覚的に表現したものです。
  実際の音声処理は寸法・座標・吸収係数・残響時間などの数値パラメータに基づいて行われます。
  ルーム形状の選択は初期反射パターンとFDNチューニングに影響しますが、
  幾何学的なレイトレーシングによる物理シミュレーションではありません。

- **処理品質**: 処理品質 設定を 高品質 にすると精度が向上しますが、CPU負荷が増加する場合があります。
  多数の音声アイテムに適用する場合はご注意ください。

- **アルゴリズムの変更**: 処理品質 やその他の設定変更は、新しいアイテムにエフェクトを追加した際に適用されます。
  既存のアイテムへの即時反映については、エフェクトを一度削除して再追加してください。

- **プリセットファイルの保存場所**: プリセットはプラグインの実行ファイルと同じディレクトリ内の
  `presets` フォルダに保存されます。YMM4のプロジェクトファイルには保存されません。

- **インターネット通信**: 本プラグインは起動時に GitHub API へのアクセスによるアップデート確認を行います。
  これ以外の目的での通信は行いません。

- **プロジェクトのバックアップ**: 本プラグインを使用する前に、YMM4プロジェクトファイルのバックアップを
  作成することを強く推奨します。

## 技術的な実装詳細

- **EarlyReflectionEngine**: 音源の鏡像点から最大24本の反射パスを生成します。
  反射ごとの距離・減衰・パンニングを計算し、独立した2本のデラインで左右チャンネルを処理します。
- **FeedbackDelayNetwork**: 素数ディレイ長を持つ8ラインのFDNと
  ハダマール行列混合により、音楽的に自然な後期残響を生成します。
  ルーム形状に応じてディレイスケールと帰還ゲインを調整します。
- **ゼロアロケーション設計**: `ArrayPool<float>` によるバッファの再利用でGCヒープ割り当てを最小限に抑えます。
- **アトミックファイル書き込み**: プリセット保存は一時ファイル経由の原子的書き込みと
  `.bak` バックアップにより、データ破損を防ぎます。
- **トースト通知**: クールダウン管理付きのトースト通知システムにより、
  同一内容の通知が連続して表示されることを防ぎます。
- **ソフトウェア3Dレンダリング**: WPF の `DrawingVisual` を用いたソフトウェアラスタライザーにより、
  パースペクティブ投影・ポリゴン裏面カリング・近平面クリッピングを実装しています。
- **自動アップデート通知**: GitHub Releases API を監視し、新バージョンが公開されている場合に
  3Dエディタのヘッダーにアップデートボタンを表示します。
